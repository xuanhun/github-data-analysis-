# Nano Banana Prompts：Star数星空轨迹图

## 一、基础信息

**项目名称**：GitHub Star数星空轨迹可视化
**工具**：Nano Banana
**目标**：将GitHub项目的Star数据转化为情感化的星空轨迹图

## 二、核心创意描述

创建一个沉浸式的3D星空场景，将GitHub项目的每个Star转化为夜空中的一颗星星，通过星星的位置、亮度、颜色和动画效果，展现项目从诞生到现在的Star增长历程，让用户感受到开源项目的成长与魅力。

## 三、详细视觉元素要求

### 1. 整体场景
- **风格**：唯美、浪漫的太空夜景，带有深邃的宇宙背景
- **视角**：默认采用3D透视视角，用户可以自由旋转、缩放和平移
- **背景**：渐变的深蓝色到黑色的宇宙背景，点缀着极远处的微弱星点
- **氛围**：带有淡淡的星云效果，营造出梦幻的太空氛围

### 2. Star数的艺术化表现
- **星星形态**：每个Star表现为一个发光的圆形粒子
- **大小变化**：星星大小从2px到8px不等，重要里程碑Star（如1k、10k、100k）更大
- **亮度变化**：星星亮度随重要程度变化，普通Star半透明，里程碑Star明亮刺眼
- **颜色编码**：
  - 亚洲用户：温暖的黄色调
  - 欧洲用户：优雅的蓝色调
  - 美洲用户：热情的红色调
  - 非洲用户：神秘的紫色调
  - 大洋洲用户：清新的绿色调

### 3. 时间维度表现
- **空间布局**：将时间轴映射到Z轴，早期Star位于远处，新Star位于近处
- **时间轴**：底部显示可交互的时间轴滑块，用户可以拖动查看不同时期的星空
- **动画效果**：
  - 默认播放模式：星星从远处（过去）向近处（现在）缓缓移动
  - 里程碑动画：重要Star出现时，播放向外扩散的波纹效果
  - 时间流逝：随时间推移，星空逐渐变得更加璀璨

### 4. 交互元素
- **鼠标悬停**：鼠标悬停在星星上时，显示以下信息：
  - Star获取时间
  - 用户GitHub头像（迷你版）
  - 用户地理位置
  - （可选）用户留言或祝福
- **点击效果**：点击星星时，放大显示详细信息，并高亮显示该用户的Star时间在时间轴上的位置
- **时间轴控制**：
  - 播放/暂停按钮
  - 快进/慢放控制
  - 直接跳转到特定时间点

## 四、技术实现细节

### 1. 数据处理要求
- **数据输入**：JSON格式的Star数据，包含以下字段：
  ```json
  [
    {
      "timestamp": "2023-01-01T12:00:00Z",
      "user": "username",
      "location": "China",
      "is_milestone": false,
      "milestone_value": null
    },
    // 更多Star数据...
  ]
  ```
- **数据映射**：
  - 时间戳 → Z轴位置（过去→远处，现在→近处）
  - 地理位置 → 颜色编码
  - 是否里程碑 → 大小和亮度

### 2. 性能优化建议
- **粒子数量控制**：当Star数超过100k时，采用分级渲染（远处星星简化为点）
- **Lod系统**：根据视角距离自动调整星星的渲染精度
- **动画优化**：使用GPU加速的粒子系统，确保流畅的动画效果

### 3. 响应式设计
- **桌面端**：完整的3D交互体验
- **移动端**：简化的2D视角，优化触摸交互

## 五、情感化设计要点

### 1. 情感基调变化
- **早期阶段**：稀疏的星星，缓慢的动画，营造孤独但充满希望的氛围
- **成长阶段**：星星逐渐增多，动画节奏加快，传递喜悦和兴奋
- **爆发阶段**：星星密集出现，播放激动人心的动画，表达成就和自豪
- **成熟阶段**：稳定的星空，明亮的核心区域，展现自信和从容

### 2. 音效设计（可选）
- **背景音乐**：轻柔的太空音乐，随星空密度变化节奏
- **交互音效**：
  - 鼠标悬停：微弱的星光音效
  - 点击星星：清脆的提示音
  - 里程碑达成：庄严的成就音效

## 六、示例应用：Vue.js项目Star可视化

假设我们要可视化Vue.js项目的280k+ Star：

1. **数据准备**：获取Vue.js的所有Star数据，标记重要里程碑（1k, 10k, 50k, 100k, 200k, 250k）
2. **颜色分配**：根据用户地理位置分配颜色
3. **动画设置**：从2014年（Vue.js诞生）开始播放，到2023年结束
4. **交互设计**：允许用户查看不同时期的Star增长情况，点击里程碑Star查看详细信息

## 七、输出要求

1. **文件格式**：支持HTML/JS输出，可直接在浏览器中运行
2. **代码质量**：结构清晰，注释完整，便于后续修改和扩展
3. **文档**：提供详细的使用说明和配置指南
